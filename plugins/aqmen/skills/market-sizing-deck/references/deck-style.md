# Aqmen Deck Style (shared)

The design system for aqmen's **CDD deliverable decks** — the editable `.pptx`
"Discussion Materials" that get delivered to clients. It defines the palette,
type, slide chrome, and chart conventions so every deck reads as one house voice,
and it documents the `aqmen_deck.py` builder that enforces them.

> Shared verbatim across all three aqmen **deck** skills. Edit the canonical copy
> in `shared/` and re-run `scripts/sync-shared.mjs`.

Decks carry the same analytical bar as the HTML reports — read
`report-standards.md` (voice, base-first, sources & confidence) and
`report-data.md` (how to pull the analysis fully from the aqmen MCP). This file
governs **form**; those govern **substance**.

## How decks are built

Decks are **real, editable PowerPoint files** generated with `python-pptx` via
the `aqmen_deck.py` module in this folder. Charts are **native PowerPoint charts**
(not images), so a consultant can open the deck and keep editing. Never
hand-author slide XML and never paste chart screenshots — always go through the
builder so the house style stays consistent.

Prerequisite: `pip install python-pptx` (the skill installs it if missing).

Page format is **16:9, 13.333in × 7.5in** — identical to the reference CDD deck.

### Branding is automatic

`aqmen_deck.Deck()` **self-brands from blank** — no base file needed. On
construction it injects the aqmen theme into the deck's theme XML (brand colour
scheme + **Montserrat** major/minor fonts, so native charts and text inherit the
brand) and bakes the **wordmark** onto the slide master (it appears on every
content slide; the builder suppresses it on the cover and full-bleed dividers,
which draw their own).

### The starter templates

Each deck skill ships a **populated starter template** — a full, styled example
deck for that module, modelled on the reference CDD deliverable:

- `references/market-sizing-deck-template.pptx`
- `references/competitive-landscape-deck-template.pptx`
- `references/company-analysis-deck-template.pptx`

Use it three ways: **(a)** open it to see the exact house style for the module;
**(b)** copy/edit its slides directly as a manual starting point; **(c)** build on
it programmatically with `Deck(template="…/<module>-deck-template.pptx")` — the
builder clears the example slides but keeps the theme, master, and layouts, so
your generated deck starts clean and on-brand.

The templates are generated by `scripts/build-templates.py` (which drives this
same builder). To retune them, edit that script or `aqmen_deck.py` and re-run,
then re-sync. Regenerating keeps the templates and live output identical in style.

## Palette (aqmen brand, from the theme)

| Token | Hex | Use |
| --- | --- | --- |
| Navy | `#03045E` | Titles, headlines, wordmark, primary data series |
| Navy deep | `#020462` | Exec-summary band, section dividers, active agenda row |
| Ink | `#302E2E` | Body text |
| Blue | `#0728A3` | Secondary data series, subtitles, links |
| Azure | `#2E6FD6` | The accent — DRAFT tag, cover bar, highlights |
| Steel | `#6BAED6` | Light steel blue — mid chart series |
| Cyan | `#ADE8F3` | Chart fills, secondary bands (e.g. SAM) |
| Cyan muted | `#D6EEF5` | Highlight rows, active agenda sub-item |
| Green | `#00FF7F` | Positive deltas only — use very sparingly |
| Grey | `#8A8A8A` | Source lines, page numbers, captions |

The palette is **blue-dominant**: navy leads every slide, supported by a coherent
ramp of blues, with azure as the *single* accent. Categorical chart series follow
the blue ramp: navy → blue → azure → steel → cyan → grey.

## Type

- **Montserrat** throughout — SemiBold/Bold for headlines and labels, Medium for
  body. `Arial` is the portable fallback (named in the file; PowerPoint
  substitutes if Montserrat is absent).
- Headline (the slide's so-what sentence): ~24pt navy bold, max two lines.
- Body bullets: ~12pt ink. Takeaways rail: ~11pt. Captions/source: ~8pt grey.

## Slide chrome (on every content slide)

- **`DRAFT`** tag top-left, azure — while the deck is not final. Turn off with
  `Deck(draft=False)`.
- **Section eyebrow** top-right, navy — `("Market Overview", "TAM")` renders as
  **Market Overview:** *TAM*.
- **Aqmen wordmark** bottom-left, navy.
- **Source line** bottom-left next to the wordmark — `Source: …`. Every slide
  with data carries one.
- **Page number** bottom-right, grey. The title slide is not paginated.
- **`ILLUSTRATIVE` / `NON-EXHAUSTIVE`** boxed tag under the headline when a chart
  is directional rather than measured.

## Slide taxonomy (the builder methods)

Every slide leads with the **so-what** as its headline, then supports it. Prefer
the right-hand **Key Takeaways** rail on analytical slides — it's where the
reader looks first.

| Method | Slide | When |
| --- | --- | --- |
| `title_slide` | Cover | First slide — title, subtitle, year |
| `agenda` | Agenda / section marker | Section transitions; highlight the `active` section and, optionally, active sub-item |
| `section_divider` | Full-bleed navy break | Heavier part breaks (e.g. Appendix) |
| `executive_summary` | Banded matrix | The one-page answer — 3–4 labelled row blocks (Context / Market / Competitive) |
| `content_slide` | Headline + body + rail | The workhorse text slide; body `Bullet`s left, `takeaways` right |
| `chart_slide` | Headline + native chart + rail | Column / bar / line / stacked quantitative exhibits |
| `marimekko_slide` | Variable-width stacked ("Mekko") | Flagship market-sizing visual — width = one dimension (e.g. region size), height = another. Segments labelled `Whitespace` render **hatched** over a solid `SAM` (TAM = SAM + whitespace) |
| `driver_tree_slide` | Expression / driver tree | Flagship market-sizing structure — the market **expression** → its **variables** (with formulas) → leaf **drivers**, joined per-parent with ×/+ operators and certainty dots. Also reused for the **WACC** build in company analysis. Keep to variables/expressions; note segmentation via `note`, don't explode every segment |
| `positioning_matrix_slide` | 2-axis archetype map | Competitive framing — each archetype is the **bounding box** of its players' `points` (+ pad), labelled, with unlabelled dots per player. Names live in the revenue build, not the boxes |
| `revenue_build_slide` | Player revenue build | **Flagship competitive-landscape visual** — players grouped by archetype, each `revenue × % addressable = market revenue` (bar), summed bottom-up to a market total. Revenue-only players supported |
| `heatmap_slide` | Sensitivity matrix | DCF WACC × terminal-growth (or any two-driver) grid, light→navy shaded, base-case cell outlined |
| `harvey_matrix_slide` | Harvey-ball comparison grid | Competitive benchmark — criteria × players, fills 0–1 |

See each skill's `*-deck-structure.md` for the required slide order per module.

## Chart rules

- **Native charts only** (`chart_slide` / `marimekko_slide`) — editable, brand-
  coloured automatically from the palette, Montserrat labels, legend at the
  bottom, light gridlines. Data labels on by default; keep them for decision
  numbers.
- No 3-D, no chartjunk, no gratuitous colour. One idea per chart. Title the chart
  with its unit (e.g. *"CDD Market TAM by Region, 2024 $B"*).
- Label the **so-what** in the headline, not the chart title. The chart proves
  the headline.
- Mark directional exhibits `illustrative=True`; never present a modelled shape
  as measured.

## Saving & exporting

`Deck.save("<name>.pptx")` writes the file. If the aqmen MCP connector is
available, offer to store the `.pptx` back in the project's Files as an artifact
(it downloads rather than rendering inline). A PDF can be produced from the
`.pptx` with LibreOffice headless if the user wants one:
`soffice --headless --convert-to pdf <name>.pptx`.

## Non-negotiables

- Real editable `.pptx` via `aqmen_deck.py`; native charts, never screenshots.
- Montserrat on the blue-dominant navy palette; navy leads, azure is the only
  accent.
- So-what headline on every slide; Key Takeaways rail on analytical slides.
- Every data slide carries a source line; directional exhibits tagged
  ILLUSTRATIVE. Never invent numbers — pull them from the analysis.
- End with a **Sources & confidence** treatment consistent with
  `report-standards.md`.
